RDF-star Test Suite

This manifest loads additional manifests for specific behavior tests for RDF-star

This is an HTML version of a test manifest. Alternate versions of the manifest may be found at manifest.ttl, and manifest.jsonld.

The RDF-star Test Suite is a set of tests that can be used to verify Processor conformance.

General instructions for running the RDF-star Test suites

Tests are run broadly the same as those for relevant test suites for Turtle, SPARQL, and RDF 1.1 Entailment.

Running tests

The top-level manifest references the specific test manifests, which in turn reference each test associated with a particular behavior. This includes the following test manifests:

  • Notation-3 Syntax tests,
  • Semantics tests,
  • SPARQL Syntax tests,
  • SPARQL Evaluation tests,
  • TriG Syntax tests,
  • TriG Evaluation tests,
  • Turtle Syntax tests,
  • Turtle Evaluation tests,

Implementations create their own infrastructure for running the test suite. In particular, the following should be considered:

  • See general guidance for running N-Triples tests.
  • See general guidance for running SPARQL 1.1 tests.
  • See general guidance for running Turtle tests.
  • See general guidance for running Semantics tests.

See the implementation report for guidance on reporting test results using the EARL format.

Contributing Tests

If you would like to contribute a new test or a fix to an existing test, please submit a Pull Request using either the GitHub web interface, the steps below with a git client, or your preferred equivalent.

  1. Clone the git repository: https://github.com/w3c/rdf-star
  2. Create a new branch.
  3. Make your changes, commit to your branch and push to your clone.
  4. Create a Pull Request in the RDF-star repository to merge into the default ("main") branch.
